@Codebase
Context: จากภาพ UI ที่ฉันให้มา ฉันรู้สึกว่าสัดส่วนขององค์ประกอบดูผิดสังเกตอย่างชัดเจน การ์ดปฏิทินดูโล่งเกินไป การ์ดรายละเอียดดูหนาแน่นเกินไป และVisual Hierarchy ไม่ชัดเจน

ให้คุณสวมบทบาทเป็น Senior UX/UI Designer และ Senior Frontend Developer ช่วยวิเคราะห์ปัญหา UX/UI จากภาพนี้อย่างละเอียด และเสนอแผนการปรับปรุงเพื่อทำให้หน้า Dashboard นี้มีสัดส่วนที่สอดคล้องกันอย่างสมเหตุสมผล กำจัดความซ้ำซ้อนของข้อมูล และทำให้คล้ายกับแอป Flo มากที่สุด



1. **การจัดวางใหม่ (Layout Redesign):** เปลี่ยนการจัดวางจากการ์ดบน-ล่างขนาดเท่ากัน เป็นการจัดวางแบบ Master-Detail ที่สมดุล:

    * **ทางเลือก (ซ้าย-ขวาแบบสมดุล):** ทำให้การ์ดปฏิทินกว้างขึ้นและสูงขึ้นทางซ้าย และการ์ดรายละเอียดแคบลงทางขวา (ถ้าคิดว่าหน้า Dashboard นี้สามารถรองรับหน้ากว้างได้)

2. **ขยายขนาดปฏิทิน:** ในการ์ด "Cycle calendar" ให้ขยายขนาดของปฏิทิน March 2026 อย่างมาก ทำให้ตัวเลขและวันมีขนาดใหญ่ขึ้น จัดให้อยู่ตรงกลางของพื้นที่การ์ด และลดพื้นที่ว่างโดยรอบ

3. **จัดกลุ่มและขยายขนาดรายละเอียดรอบเดือน:** ในการ์ด "Cycle details":

    * ขยายขนาดข้อมูลวันที่ (2026-03-17 -> 2026-03-21) ให้ใหญ่ขึ้นมากและเด่นขึ้น

    * เพิ่ม Spacing ระหว่างรายการรอบเดือน (Items)

    * ขยายขนาดปุ่ม "Edit", "Delete" ให้ใหญ่ขึ้นและกดง่ายขึ้น จัดวางให้อยู่ใกล้กับข้อมูลวันที่ (เช่น จัดเรียงเป็นคอลัมน์: ข้อมูล | ปุ่ม) เพื่อกำจัดพื้นที่ว่างตรงกลาง

4. **ปรับสไตล์องค์ประกอบอื่นๆ:** ปรับ Segmented Control ("History", "Trends") ให้มีสัดส่วนที่ดูดีและคล้าย Flo มากขึ้น และตรวจสอบการใช้ Typography ให้สอดคล้องกัน
